# <a name="lab3"></a> Лабораторная работа 3. Разработка синтаксического анализатора (парсера)

#### Цель работы: Изучить назначение и принципы работы синтаксического анализатора в структуре компилятора. Спроектировать грамматику, построить соответствующую схему метода анализа грамматики и выполнить программную реализацию парсера с нейтрализацией синтаксических ошибок методом Айронса. Интегрировать разработанный модуль в ранее созданный графический интерфейс языкового процессора.


#### Постановка задачи: Разработать синтаксический анализатор (парсер) в соответствии с индивидуальным вариантом курсовой (расчетно-графической) работы, интегрировать его в приложение из лабораторной работы №1 и обеспечить наглядный вывод результатов анализа.

- Реализовать алгоритм нейтрализации синтаксических ошибок методом Айронса.

Выходные данные:
- При успешном анализе корректной строки — сообщение об отсутствии ошибок.

### Классификация грамматики (по Хомскому)
Разработанная грамматика является автоматной, то есть относится к грамматикам 3 типа по классификации Хомского.
A → aB
В правой части каждого правила находится один терминальный символ и не более одного нетерминального символа. При этом нетерминальный символ, если он присутствует, расположен справа.

### Метод анализа
- Так как разработанная грамматика является автоматной, синтаксический анализ выполняется методом Айронса, основанным на работе конечного автомата.
Каждое состояние автомата соответствует ожиданию определённой лексемы грамматики:
- DECLARE → identifier → CONSTANT → INTEGER → := → number → ;
- Анализатор последовательно просматривает список лексем, полученный после лексического анализа, и проверяет соответствие текущей лексемы ожидаемому элементу грамматики. Если лексема распознана верно, выполняется переход к следующему состоянию.
- При обнаружении синтаксической ошибки в программе используется метод нейтрализации ошибок Айронса. Его суть заключается в том, что после обнаружения ошибки анализатор не прекращает работу, а пропускает ошибочные лексемы до ближайшей точки синхронизации, после чего продолжает разбор.

###Диагностика и нейтрализация синтаксических ошибок

Для диагностики и нейтрализации синтаксических ошибок в программе реализован механизм восстановления разбора, основанный на идее метода Айронса.

Основная идея состоит в том, что при возникновении синтаксической ошибки анализатор не завершает работу сразу, а:

фиксирует ошибочный фрагмент;
определяет местоположение ошибки;
формирует сообщение об ошибке;
выполняет восстановление разбора;
продолжает анализ оставшейся части входной строки.

В данной работе синтаксический анализ выполняется для конструкции объявления константы вида:

DECLARE identifier CONSTANT INTEGER := number ;
- Синтаксический анализатор последовательно проверяет наличие ключевого слова DECLARE, идентификатора, ключевого слова CONSTANT, типа INTEGER, оператора присваивания :=, числового значения и символа ;.
- При обнаружении ошибки вызывается метод AddError(), который сохраняет неверный фрагмент, позицию ошибки и описание ошибки. Например, если вместо ожидаемого элемента найден другой токен, формируется сообщение вида: Ожидалось 'DECLARE', найдено 'product_name'. Для проверки отдельных элементов конструкции используются методы:
1. IsKeyword() — проверка ключевого слова;
2. IsIdentifier() — проверка идентификатора;
3. IsOperator() — проверка оператора;
4. IsNumber() — проверка числа;
5. IsSemicolon() — проверка символа ;
- Восстановление после ошибки выполняется путём пропуска некорректных лексем до ближайшей точки синхронизации. В программе для этого используется метод SkipToNumberOrSemicolon(), который применяется при ошибке в операторе присваивания и пропускает лексемы до числа или символа ;. Это позволяет не создавать лишние ошибки после одного неверного фрагмента. В качестве точек синхронизации используются ключевые элементы конструкции:
  - DECLARE — начало объявления;
  - CONSTANT — признак объявления константы;
  - INTEGER — тип данных;
  - := — оператор присваивания;
  - number — числовое значение;
  - ; — конец объявления.
Также в программе предусмотрена обработка лишнего текста после символа ;. Если после завершения объявления остаются дополнительные лексемы, анализатор формирует ошибку о том, что ожидался конец объявления.
- Такой подход позволяет:
  - не останавливать анализ после первой ошибки;
  - выводить информацию о найденных синтаксических ошибках;
  - указывать ошибочный фрагмент и его местоположение;
  - уменьшать количество каскадных ошибок;
  - продолжать разбор после восстановления по точкам синхронизации.

- Результаты синтаксического анализа выводятся в отдельную таблицу ошибок. Для каждой ошибки отображается неверный фрагмент, местоположение и описание, а общее количество ошибок выводится в специальную метку на форме.
